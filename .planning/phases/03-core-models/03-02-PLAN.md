<objective>
Build the pro-forma recoveries page with interactive EV slider and color-coded recovery heatmap, plus the pro-forma debt service schedule page with per-tranche interest visualization.

Purpose: Recoveries and debt service are the two analytical views that show what happens AFTER restructuring -- how much each creditor class gets back (recoveries) and whether the restructured entity can service its new debt (debt service). The EV slider on recoveries lets analysts explore "what if EV is X?" scenarios instantly.

**Create `site/js/pf-recovery-model.js`:**

IIFE module, ES5 with `'use strict'`, `var` throughout.

**Data Loading:**
- `AMC_UTILS.DataLoader.fetch('pf-recoveries.json')` and `AMC_UTILS.DataLoader.fetch('cap-table.json')`

**Recovery Table (renderRecoveryTable function):**
- Render the `recoveries` array as an HTML table
- Columns: Tranche Name, Lien (from entityCapTables data), Face Value, Odeon %, Odeon $, AMC %, AMC $, Muvico %, Muvico $, Total %, Total $
- Format percentages with `fmtPct(n * 100)` (they are decimals like 0.887)
- Format dollar amounts with `fmt()`
- Add a totals row at the bottom from `totals` object
- Color-code the total recovery % cells: >=80% green (#22c55e), 40-80% yellow (#eab308), <40% red (#ef4444)

**EV Slider Interaction (updateForEV function):**
- When the user moves the EV slider, recalculate recovery amounts assuming pro-rata scaling.
- The JSON data shows recoveries at a specific EV (evBreakdown total = 2176). When the user selects a different EV:
  - Scale the entity EV breakdown proportionally: each entity's EV = (entity_original_EV / total_original_EV) * new_total_EV
  - For each tranche, recalculate: new_recovery_amt = tranche_original_recovery_amt * (new_total_EV / original_total_EV), capped at face value
  - Actually, a more accurate approach: use a waterfall-style allocation. The pf-recoveries.json shows recovery at one EV point. For the slider, generate multiple EV scenarios by scaling linearly and capping each tranche at its face. Walk the capital structure top-to-bottom (by lien priority): allocate EV to senior tranches first until face is covered, then spill to junior.
  - Use the tranche order from the recoveries array (which is already sorted by seniority: Term Loans first, Sub notes last)
  - For the waterfall: total claims = sum of all face values. For a given EV, allocate top-down. Each tranche gets min(face, remaining_EV). Recovery % = allocated / face.
  - The entity-level breakdown (Odeon/AMC/Muvico split) should use the original proportions from evBreakdown.

- Update the recovery table, heatmap, key metrics, and chart with new values

**Heatmap (renderHeatmap function):**
- Generate a grid of recovery scenarios: 10-12 EV values from $500M to $5,000M in ~$500M steps
- Rows = tranches (10 tranches from recoveries array)
- Columns = EV scenarios
- Cell value = recovery % at that EV for that tranche (use waterfall allocation)
- Cell color: gradient from red (0%) through yellow (50%) to green (100%). Use inline CSS background-color.
- Highlight the column closest to the current EV slider value with a border or brighter background
- Use compact formatting: show percentage as integer (e.g., "89%") to fit in cells

**Recovery Waterfall Chart (renderRecoveryChart function):**
- Horizontal stacked bar chart using Chart.js
- One bar per tranche
- Stacked segments: Odeon recovery (blue), AMC recovery (gray), Muvico recovery (orange)
- Use `AMC_UTILS.ChartRegistry.set('pf-recovery', ...)` for lifecycle management
- Show face value as a reference line or total bar length

**Entity Cap Table Section (renderEntityCapTables function):**
- Collapsible/expandable section (hidden by default, toggle button to show)
- For each entity in entityCapTables: render a mini table with columns: Tranche, Lien, Face, Price, Market, Coupon, Cash Interest, Maturity, YTM
- Show summary rows (Total Debt, Net Debt) with leverage metrics (Debt/EBITDA face, Debt/EBITDA market)

**Key Metrics Bar:**
- On each update: Total EV, Weighted Avg Recovery %, Senior (1L) Recovery %, Junior Recovery %

**Slider Event Listeners:**
- EV slider `input` event -> updateForEV()
- View toggle click -> show/hide table vs heatmap
- Preset buttons -> set slider value, trigger updateForEV()

**Initialize:** Load data, render at default EV, set up listeners.

**Create `site/js/debt-service-model.js`:**

IIFE module, ES5, `'use strict'`, `var` throughout.

**Data Loading:**
- `AMC_UTILS.DataLoader.fetch('pf-ds.json')`

**Summary Table (renderSummaryTable function):**
- Compute aggregate totals across tranches 0-7 (skip tranche 8 which is the consolidated/total row in the JSON) for each period:
  - Total Starting Balance (sum of startingBalance[i] across tranches 0-7)
  - Total Interest Expense (sum of interestExpense[i])
  - Total Principal Payments (sum of absolute borrowingsRepayments[i], treating negative as repayments)
  - Total Debt Service = Interest + Principal
  - Total Ending Balance
- Render as HTML table: rows = metrics, columns = periods (2026-2032)
- Add a "PIK Accrual" row showing sum of pikToggle values (some tranches have pikToggle arrays)
- Format with `AMC_UTILS.fmt()`

**Detail Table (renderDetailTable function):**
- For each of the 8 real tranches (indices 0-7):
  - Header row: Tranche name, interest rate, maturity year
  - Sub-rows: Starting Balance, Interest Expense, PIK Toggle (if applicable), Principal Payment, Ending Balance
  - Values for each period (2026-2032)
- Use collapsible rows: show only tranche header + total interest by default, expand to show all sub-rows on click
- Color tranches by type: Term loans (red tones), PIK notes (orange/yellow), fixed rate notes (blue/gray)
- Handle the 6/8% PIK Toggle tranche specially -- it has a `toggleCashPik` array indicating cash vs PIK periods

**Interest Stacked Bar Chart (renderInterestChart function):**
- Stacked vertical bar chart: X-axis = periods, Y-axis = interest expense ($M)
- One dataset per tranche (8 tranches), each with its own color
- Use `AMC_UTILS.ChartRegistry.set('ds-interest', ...)` for lifecycle
- Tooltip shows per-tranche breakdown for each period
- Format Y-axis with `AMC_UTILS.fmt()`

**Balance Area Chart (renderBalanceChart function):**
- Stacked area chart showing total outstanding balance by tranche over time
- X-axis = periods, Y-axis = balance ($M)
- Shows how debt declines as maturities hit (Odeon notes 2027, 15% PIK 2029, etc.)
- Use `AMC_UTILS.ChartRegistry.set('ds-balance', ...)` for lifecycle

**Key Metrics:**
- Year 1 total interest: sum of all tranche interestExpense[0]
- Year 1 total debt service: interest + principal for period 0
- Peak interest year: period with highest total interest
- Total principal over projection: sum of all abs(borrowingsRepayments) across all tranches and periods

**View Toggle:**
- Summary view: show summary table + both charts
- Detail view: show detail table + both charts

**Initialize:** Load data, render summary view, set up toggle listener.
