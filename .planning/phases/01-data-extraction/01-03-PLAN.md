- Cap Table: check sum of `face` values per entity against any total row in Cap Table sheet

**Reconciliation assertions to implement:**

1. **IS Gross Profit check**: For each period in is.json:
   - `gross_profit[i] == round(sales[i] + cogs[i], 1)` (COGS is negative in IS)
   - Tolerance: `abs(extracted['is']['grossProfit'][i] - expected) < 0.2`
   - On failure: `raise AssertionError(f"IS Gross Profit mismatch at period {period}: extracted {extracted_val}, expected {expected_val}")`

2. **IS EBIT check**: For each period in is.json:
   - `ebit[i] == round(grossProfit[i] + sga[i], 1)` (SGA is negative)
   - Same tolerance and raise pattern

3. **BS integrity check**: For each period in bs.json:
   - Accounting identity: Total Assets == Total Liabilities + Total Equity
   - `bs = extracted.get('bs', {})`
   - `for i, period in enumerate(bs.get('periods', [])):`
   - `    expected = bs['totalLiabilities'][i] + bs['totalEquity'][i]`
   - `    if not (abs(bs['totalAssets'][i] - expected) < 0.5):`
   - `        raise AssertionError(f"BS integrity check failed at period {period}: totalAssets={bs['totalAssets'][i]}, totalLiabilities+totalEquity={expected}")`
   - Tolerance: 0.5 (BS totals can accumulate more rounding than IS line items)

4. **Cap Table total check**: Sum of `face` values for each entity section (Muvico, Odeon, AMC separately) should match any total row found in cap-table.json.
