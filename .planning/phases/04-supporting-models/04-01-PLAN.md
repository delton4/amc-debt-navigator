**JS model (`site/js/revenue-model.js`):**
IIFE pattern with `'use strict'`. Use `var` throughout (ES5 convention). Use `window.AMC_UTILS` for all formatters, ChartRegistry, DataLoader.

1. **Data loading:** `AMC_UTILS.DataLoader.fetch('revenue-build.json')` then call `init(data)`.

2. **Data filtering logic:**
   - Annual view: filter `data.periods` to entries that do NOT contain 'Q' (e.g., "2020", "2021", ..., "2030") -- these are the annual totals already in the data
   - Quarterly view: filter to entries that DO contain 'Q' (e.g., "2021Q1", "2021Q2", ..., "2030Q4")
   - Store current view in a module-level `var currentView = 'annual'`

3. **Main stacked bar chart** (`canvas#revenue-chart`):
   - Chart type: `'bar'` with `stacked: true` on both x and y axes
   - 6 dataset stacks for the filtered view: usAdmissions (color #3b82f6), usFoodAndBeverage (#60a5fa), usOtherTheatre (#93c5fd), intlAdmissions (#f59e0b), intlFoodAndBeverage (#fbbf24), intlOtherTheatre (#fcd34d)
   - Stack groups: US segments in stack "US", International segments in stack "Intl" -- this creates a grouped stacked bar (US stack + Intl stack side by side per period)
   - Actually, use a single stack per bar position (all 6 segments stacked into one bar per period) for cleaner display. Use borderColor to visually separate US (blue tones) from Intl (amber tones).
   - For projected periods (where `data.isProjected[i]` is true), reduce bar opacity to 0.7 to visually distinguish forecast from actuals
   - X-axis labels: the filtered period strings
   - Y-axis: "$M" label, ticks formatted with commas
   - Tooltip: show segment name + value formatted with `AMC_UTILS.fmt()`, plus total
   - Register chart via `AMC_UTILS.ChartRegistry.set('revenue-chart', new Chart(...))`

4. **Segment mix chart** (`canvas#mix-chart`):
   - Only for annual view (hide container in quarterly view)
   - Chart type: `'bar'` with stacked:true, but data as percentages (US% and Intl% of total)
   - Two datasets: "US" (blue #3b82f6) and "International" (amber #f59e0b)
   - Each bar sums to 100%. Values computed as `totalUsRevenue[i] / totalRevenue[i] * 100`
   - Y-axis max 100, ticks show "%"
   - Register via `AMC_UTILS.ChartRegistry.set('mix-chart', new Chart(...))`

5. **Growth table:**
   - Build rows for annual view only: Period | Total Revenue | YoY Growth % | US Revenue | Intl Revenue | US Share % | Intl Share %
   - YoY growth = `(current - prior) / prior * 100`, format with `AMC_UTILS.fmtPct()`
   - Color growth cells: green for positive, red for negative via `AMC_UTILS.colorVal()`
   - For quarterly view, show a simpler table with just period + total revenue

6. **Summary metrics** (populate `#summary-metrics`):
   - Find FY2024 and FY2030 annual indices in `data.periods`
   - FY2024 Total Revenue formatted with `AMC_UTILS.fmt()`
   - FY2030E Total Revenue formatted with `AMC_UTILS.fmt()`
   - CAGR 2024-2030: `((rev2030/rev2024)^(1/6) - 1) * 100`, show as percentage
   - US % of Total for latest actual year

7. **Toggle handler:**
   - Expose `window.setRevenueView = function(view)` for annual/quarterly toggle buttons
   - On toggle: update `currentView`, re-filter data, destroy and recreate charts via ChartRegistry, update growth table, show/hide mix chart container
   - Update active button class (add/remove 'active' on `tab-btn` elements)

Key conventions:
- Use `var` not `let`/`const` (ES5 codebase)
- All charts through `AMC_UTILS.ChartRegistry.set()`
- All data through `AMC_UTILS.DataLoader.fetch()`
- No `Chart.defaults.*` mutations (already handled by amc-utils.js)
