**JS model (`site/js/comps-model.js`):**
IIFE pattern with `'use strict'`. Use `var` throughout (ES5 convention). Use `window.AMC_UTILS` for all formatters, ChartRegistry, DataLoader.

1. **Data loading:**
   ```
   Promise.all([
     AMC_UTILS.DataLoader.fetch('comps.json'),
     AMC_UTILS.DataLoader.fetch('odeon-football-field.json')
   ]).then(function(results) { init(results[0], results[1]); });
   ```

2. **Comps Table Rendering (`renderCompsTable`):**
   - Store `compsData` at module scope for sorting
   - Store current `sortField` and `sortDir` (asc/desc toggle on re-click)
   - Render `compsData.companies` into `#comps-table tbody`:
     - Each row: company name, ticker, price (fmtDollar), marketCap (fmtInt), ev (fmtInt), evEbitda2026 (fmtX), evEbitda2027 (fmtX), evRevenue2026 (fmtX), evRevenue2027 (fmtX), beta (2 dp)
     - **AMC row highlighting:** If `company.ticker === 'AMC'`, add class `amc-highlight` to the `<tr>`
   - Render statistics rows in `#comps-stats`: a mini-table with rows for Min, Q1, Median, Q3, Max, Average showing evEbitda2026, evEbitda2027, evRevenue2026, evRevenue2027 from `compsData.statistics`

3. **Sort handler (`window.sortComps = function(field)`):**
   - Toggle sortDir if same field, else set to 'desc' (for multiples, descending default makes sense)
   - Sort `compsData.companies` by the field value
   - Re-render the table body
   - Update header indicators: append a small triangle character to the active sort column header

4. **AMC Callout (`#amc-callout`):**
   - Compute AMC premium/discount vs peer median for EV/EBITDA 2026E: `(amcMultiple - medianMultiple) / medianMultiple * 100`
   - Display text like: "AMC trades at 10.62x EV/EBITDA 2026E vs. peer median of 7.97x, a 33.2% premium reflecting distressed capital structure dynamics."
   - Use class `amc-callout` for styling

5. **Summary Metrics** (populate `#comp-metrics`):
   - Peer Median EV/EBITDA: `compsData.statistics.median.evEbitda2026` formatted with fmtX
   - AMC EV/EBITDA: AMC company's evEbitda2026 formatted with fmtX
   - Premium/(Discount): percentage vs median, colored green/red
   - Peer Count: "4 peers" (count of non-AMC companies)

6. **Football Field Chart (`canvas#football-chart`):**
   - Chart type: `'bar'` with `indexAxis: 'y'` (horizontal bars)
   - Use Chart.js native floating bar format: data values as `[min, max]` pairs
   - Each bar = one valuation method from `odeon-football-field.json.ranges`
   - Labels (y-axis): method names from ranges array. Use shorter aliases for display:
     - "LTM EV/Revenue" -> "LTM EV/Rev"
     - "LTM EV/EBITDA" -> "LTM EV/EBITDA"
     - "FWD 2026 EV/Revenue" -> "'26E EV/Rev"
     - "FWD 2026 EV/EBITDA" -> "'26E EV/EBITDA"
     - "FWD 2027 EV/Revenue" -> "'27E EV/Rev"
     - "FWD 2027 EV/EBITDA" -> "'27E EV/EBITDA"
     - "P/E" -> "P/E"
     - "DCF GG" -> "DCF (Gordon)"
     - "DCF MM" -> "DCF (Multiples)"
   - Single dataset: label "Implied EV Range ($M)"
   - Bar colors: use a consistent #3b82f6 with 0.7 alpha for comps-based methods, #22c55e with 0.7 alpha for DCF methods (last 2 bars)
   - **Annotation plugin reference line:** The football field data is Odeon-level valuations ($100-700M range). Add a vertical annotation line at the midpoint of the DCF methods: `((105.12 + 135.64) / 2 + (163.09 + 185.11) / 2) / 2`. Compute dynamically from the "DCF GG" and "DCF MM" ranges. Label it "DCF Mid" with dashed green line style using chartjs-plugin-annotation config under `plugins.annotation.annotations`.
   - X-axis: "$M" label, ticks formatted with commas
   - Tooltip: show "Method: $min - $max ($M)" on hover
   - Register via `AMC_UTILS.ChartRegistry.set('football-chart', new Chart(...))`

Key conventions:
- Use `var` not `let`/`const` (ES5 codebase)
