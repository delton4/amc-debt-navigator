```javascript
(function() {

  var fmtPct = U.fmtPct;

  // ── Waterfall computation (duplicated from pf-recovery-model.js) ──
  function computeWaterfall(totalEV, recoveries, evBreakdown, trancheFaces) { ... }

  // ── Data Load ──
  Promise.all([
    U.DataLoader.fetch('ufcf.json'),
    U.DataLoader.fetch('valuation.json'),
    U.DataLoader.fetch('wacc.json'),
    U.DataLoader.fetch('cap-table.json'),
    U.DataLoader.fetch('pf-recoveries.json'),
    U.DataLoader.fetch('is.json')

    var ufcf = results[0], val = results[1], wacc = results[2];

})();
```

Populate KPI tiles by ID. For each KPI, find element by id, set `.value` innerHTML and add appropriate color class:

| KPI ID | Source | Format | Color |
|--------|--------|--------|-------|
| kpi-ev-exit | ufcf.terminalValue.enterpriseValue | fmtDollar + "M" | neutral |
| kpi-ev-perp | ufcf.perpetuityGrowth.enterpriseValue | fmtDollar + "M" | neutral |
| kpi-equity | val.amc.scenario1["Implied Equity Value"] | fmtDollar + "M" | distress (negative) |
| kpi-share | val.amc.scenario1["Share Price"] | "$" + value.toFixed(2) | distress (negative) |
| kpi-debt | wacc.inputs.totalPrincipal | fmtDollar + "M" | neutral |
| kpi-wacc | wacc.inputs.wacc | fmtPct | distress (high) |
| kpi-cost-debt | wacc.inputs.pretaxCostOfDebt | fmtPct | distress (high) |

**renderCapStructure(cap):**

For each tranche in `cap.tranches`:
- Face formatted with `fmt()` + "M"
- Coupon as percentage
- Maturity as "MMM YYYY" (extract from date string)

- Add a small horizontal bar (CSS width proportional to face value relative to max face) for visual density
- Color-code lien column using existing tag classes (tag-1l, tag-15l, tag-2l, tag-unsec)

Use the `.data-table` class from theme.css for table styling.

**renderHeatmap(pfRec):**
Build the recovery heatmap into `#recovery-heatmap`. This is a DOM-based grid (not canvas).

1. Derive tranche face values from `pfRec.recoveries` (each recovery object's totalAmt at 100% recovery)
2. Generate EV scenarios: 11 steps from $0M to $5,000M in $500M increments (matching pf-recovery model)
3. For each EV step, run `computeWaterfall()` to get per-tranche recovery percentages
4. Build grid:
   - Header row: blank cell + EV scenario labels ("$0M", "$500M", ... "$5,000M")
   - One row per tranche: tranche name label + cells with recovery % colored by recoveryColor/recoveryBgColor
   - Cell format: show percentage as integer (e.g., "100", "67", "0") -- no % sign to save space
   - Color each cell's text with `recoveryColor(pct)` and background with `recoveryBgColor(pct)`
5. Exclude tranche 8 (6.125% AMC Unsecured consolidated total row) per Phase 3 decision

The heatmap should be horizontally scrollable if needed (overflow-x auto on container).
